# T01: Build ModeToggle component with TDD and wire into sidebar footer

**feat: add ModeToggle dropdown with dark/light/system theme switching wired into sidebar footer**

## What Happened

Built the ModeToggle component following TDD: wrote 6 tests first (red phase confirmed — module not found), then implemented the component and barrel export, then wired it into AppShell's SidebarFooter. The shadcn/ui DropdownMenu was added via `npx shadcn@latest add dropdown-menu` with the K006 Windows path fix (moved files from literal `@/` to `src/components/ui/`).

The ModeToggle renders a ghost button with Sun/Moon icons that animate on theme change (CSS rotate+scale transitions). Clicking opens a Radix DropdownMenu with three options (Light, Dark, System). Each calls `setTheme()` from the existing ThemeProvider context, which handles localStorage persistence and DOM class application.

Tests use the real ThemeProvider (via `renderWithProviders`) and verify theme changes by checking `document.documentElement.classList` and `localStorage` — no mocking of useTheme needed.
